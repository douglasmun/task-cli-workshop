Threat Actor Profile — IronLock Ransomware Group

IronLock is a financially motivated ransomware-as-a-service (RaaS) operation that has been active since mid-2024, with a documented focus on healthcare organizations across North America and Western Europe. The group operates a double-extortion model, exfiltrating patient records and administrative data prior to encrypting endpoints, then threatening public release via their Tor-hosted leak site. Attribution to a specific nation-state has not been established; infrastructure patterns suggest Eastern European operators with some affiliate recruitment conducted via Russian-language cybercrime forums.

Initial access is primarily achieved through exploitation of unpatched VPN appliances and Remote Desktop Protocol exposure, corresponding to MITRE ATT&CK techniques T1190 (Exploit Public-Facing Application) and T1133 (External Remote Services). Following initial compromise, IronLock affiliates deploy a modified version of Cobalt Strike for command-and-control (T1071.001 — Application Layer Protocol: Web Protocols) and conduct extensive Active Directory reconnaissance using BloodHound and SharpHound (T1087.002 — Account Discovery: Domain Account). Lateral movement relies heavily on pass-the-hash and Kerberoasting (T1550.002, T1558.003). Data exfiltration observed prior to encryption leverages Rclone configured to attacker-controlled cloud storage endpoints (T1567.002).

The IronLock encryptor targets Windows and ESXi environments. VMware ESXi hosts are encrypted using a Linux ELF binary delivered via SSH after credentials are obtained from vCenter. The ransom demand for healthcare targets has ranged from USD 400,000 to USD 2.1 million.

Defenders should prioritize patching internet-facing VPN and RDP, enforcing MFA on all remote access, and monitoring for BloodHound-characteristic LDAP query volumes.
