Incident Report — Supply Chain Compromise via npm Package event-stream-lite

On 2025-11-04, security researchers at a package registry monitoring service identified anomalous behavior in the npm package event-stream-lite version 2.3.1, published approximately 72 hours earlier. Analysis of the package revealed an obfuscated payload embedded in a post-install script that downloaded a second-stage binary from a remote host. The payload targeted developer environments and CI/CD pipeline runners, specifically exfiltrating SSH private keys, AWS credentials from ~/.aws/credentials, and environment variables containing CI tokens.

The malicious version 2.3.1 was published by an account that had been granted maintainer access to the legitimate project six weeks prior. The legitimate maintainer had abandoned the package in 2024, and the attacker registered a new npm account using a similar username and submitted a maintainer transfer request through the package registry's web interface. The registry approved the request without additional verification.

The package had approximately 340,000 weekly downloads at the time of compromise. Dependent packages confirmed to have pulled in the malicious version during the exposure window include build-pipeline-utils 1.9.x, webpack-event-helpers 0.7.2 through 0.7.4, and several private packages identified by affected organizations. The exposure window ran from 2025-11-01 to 2025-11-05 when the registry removed the package following researcher notification.

Detection signatures were released by several EDR vendors within 24 hours. The post-install script can be identified by the presence of a base64-encoded string matching the pattern beginning with ZXhlYyhy in the package's install hook. Affected organizations should rotate all credentials accessible from CI/CD environments that installed any package version during the exposure window, audit npm dependency lock files for unexpected version pins, and review access logs for exfiltration attempts to the C2 domain packages-registry[.]workers[.]dev.
