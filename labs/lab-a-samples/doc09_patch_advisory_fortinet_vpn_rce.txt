Patch Advisory — Critical RCE in Fortinet FortiGate SSL-VPN

Fortinet has released an emergency patch addressing CVE-2026-3122, a critical unauthenticated remote code execution vulnerability in FortiGate SSL-VPN appliances. The vulnerability carries a CVSS v3.1 base score of 9.6 and affects FortiOS versions 7.0.0 through 7.0.14, 7.2.0 through 7.2.8, and 7.4.0 through 7.4.3. The flaw is a heap-based buffer overflow in the SSL-VPN web management interface's HTTP request parsing library, reachable without authentication from the WAN interface if SSL-VPN is enabled.

Exploitation in the wild was confirmed on 2026-03-18, approximately six days after the patch was released. Observed exploitation activity includes deployment of a Perl-based webshell to the appliance filesystem at a path under /data/lib/, execution of OS commands to establish outbound reverse shells, and subsequent use of the compromised appliance as a pivot point into internal network segments. Several threat clusters, including one assessed as a nation-state actor, were observed exploiting this vulnerability within 72 hours of public disclosure.

The permanent fix is to upgrade to FortiOS 7.0.15, 7.2.9, or 7.4.4 or later, depending on the deployed branch. Fortinet has published upgrade instructions in PSIRT advisory FG-IR-26-009. If an immediate upgrade is not operationally feasible, the recommended workaround is to disable SSL-VPN entirely via the CLI. Note that disabling the web-based management interface alone is insufficient; the vulnerability is in the SSL-VPN data path, not the administrative GUI.

Organizations should audit FortiGate logs for POST requests to /remote/login containing anomalous Content-Length values, and check for unexpected files under /data/lib/ and /var/tmp/ on affected appliances. Any appliance that was exposed to the internet running a vulnerable version should be treated as potentially compromised and subject to full forensic review before being returned to service.
