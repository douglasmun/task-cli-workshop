CVE-2026-1847 — Linux Kernel Local Privilege Escalation via io_uring

A local privilege escalation vulnerability affecting the Linux kernel has been assigned CVE-2026-1847 with a CVSS v3.1 base score of 7.8 (High), vector CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H. The vulnerability exists in the io_uring subsystem, specifically in the handling of IORING_OP_PROVIDE_BUFFERS operations when a userspace process registers a buffer group with a negative buffer identifier. A race condition in the reference counting logic allows an unprivileged local user to corrupt kernel heap memory, leading to a use-after-free condition that can be leveraged to overwrite function pointers and execute arbitrary code in kernel context.

Affected kernel versions include 5.15 LTS through 6.6.29 and 6.8.0 through 6.8.7. The 6.6 stable branch received a backport fix in 6.6.30. The 6.8 branch was patched in 6.8.8. The 6.9 development branch was not affected as the buffer group registration path was refactored prior to release. Distributions shipping affected kernels include Ubuntu 22.04 with kernel packages prior to 5.15.0-118-generic, Debian 12 with kernels before 6.1.90-1, and Red Hat Enterprise Linux 9 with kernel-5.14.0-427.13.1 and earlier.

A proof-of-concept exploit was published to a public research blog on 2026-02-03, approximately 48 hours after the patch was merged to the mainline tree. Exploitation has been confirmed in containerized environments where the container runtime does not restrict io_uring via seccomp. Organizations running shared Linux hosts or container workloads should apply kernel updates immediately. As a mitigation where patching is not immediately possible, disabling io_uring system-wide via sysctl setting kernel.io_uring_disabled=1 prevents exploitation but will impact applications that rely on io_uring for high-performance I/O.
