CVE-2025-38471 — Authentication Bypass in Django REST Framework

A critical authentication bypass vulnerability has been identified in Django REST Framework versions 3.14.0 through 3.15.2. The flaw resides in the token-based authentication middleware, where a malformed Authorization header containing a null byte prefix causes the framework to skip signature validation and grant access as the first user in the database. This vulnerability is tracked as CVE-2025-38471 and carries a CVSS v3.1 base score of 9.8 (Critical), with the vector string CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H.

Exploitation requires no authentication and no user interaction. An unauthenticated attacker can send a crafted HTTP request to any endpoint protected by TokenAuthentication and gain full access to the application as an administrative user. The issue was discovered by a researcher during a black-box penetration test of a financial services portal and disclosed to the maintainers under a 90-day coordinated disclosure timeline.

Affected versions are Django REST Framework 3.14.0, 3.14.1, 3.15.0, 3.15.1, and 3.15.2. Applications using SessionAuthentication or OAuth2 token flows are not affected by this specific code path. The vulnerability has been patched in version 3.15.3, released on 2025-09-14. Operators should upgrade immediately. If an immediate upgrade is not possible, a temporary workaround is to add a middleware class that rejects any Authorization header containing non-printable characters before the request reaches the DRF authentication layer. No public exploit code was available at the time of this advisory, but the exploitation technique is trivial to reproduce from the patch diff.

Organizations running Django REST Framework in internet-facing deployments should treat this as urgent. Review application logs for anomalous access patterns, particularly requests that authenticated as high-privilege users from unexpected IP addresses in the days preceding patch application.
